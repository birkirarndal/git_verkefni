Git spurningar og svör

1. Hvað gera eftirfarandi Linux skipanir?
	cd:	breytir í home directory.
	cd ~: breytir í root directory.
	ls: sýnir directory lista.
	pwd: sýnir hvað directory maður er í.
	mkdir: býr til directory.

2. Hvað gera eftirfarandi git skipanir og hvers vegna eru þær gagnlegar?
	git clone: býr til afrit af remote repository.
	git log: sýnir commit history.
	git status: skilar virkandi branch.
	git diff: sýnir öll merge conflicts.
	git checkout: skiptir um branch.

3. Hver er munurinn á eftirfarandi git skipunum:
	a) git diff 
	b) git diff --staged 
	c) git diff commit1 commit2
a. sýnir öll merge conflicts. b. sýnir conflicts á base file. c. sýnir breytingar aður en að það er merge-að.

4. 	Hvað er útgáfustýring (e.version control)? 
útgáfustýring er system sem recordar breytingar á skrám svo að það sé hægt að fara aftur á áhveðin gerð af skránni seinna.

5.	Hverjir eru helstu kostir við að nota GIT?
git er getur haldið gömlum gerðum af skránni sem maður er að vinna í þannig ef eitthvað klikkar þá getur maður farið aftur til baka í skrána sem virkaði og síðan er hæt að vinna með morgu fólki í sömu skrá.

6.	Hversu oft telur þú að eigi að gera commit í verkefni, rökstuddu?
ég myndi seigja eftir hverja breytingu þannig maður man eftir hvaða breytingu var gerð.

7.	Hvað er "Working directory"?" "Staging area"?" og "Repository" í GIT?
repository sem er búinn til með "git init" skipun er kallaðar working directory. Staging er  skrefið áður en að það er búið að gera commit. git geymir verkefni, skrá þegar þær breytast, þessar upplýsingar geymir git í repository.

8.	Hvenær er sniðugt að nota greinar (branches)? 
þegar maður byrjar að vinna á nýrri feature.
